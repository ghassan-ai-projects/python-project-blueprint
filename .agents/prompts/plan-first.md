# Plan First

Before editing, inspect the repository and write a short plan.

Your plan must include:

- the relevant files to read first
- the files you expect to create or update
- why each file is needed
- the smallest safe implementation strategy
- why that strategy is simpler than the obvious alternatives
- what test will prove the change
- whether that test can be written before implementation
- validation commands
- risks, assumptions, or places that need human review

Do not implement until the plan is specific enough that another engineer could review it quickly.
